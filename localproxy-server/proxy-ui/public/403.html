localproxy

Error 403

localproxy encountered a 403 condition when trying to serve that request.

This is usually happens when a static directory matches the route but the route either doesn't match a file in that directory or the file has the read permission turned off.

Does your request match a file in your directory? Is that file readable by the localproxy user? Do you need to enable the directory listing (DIR) or root index fallback (RIF) settings?
